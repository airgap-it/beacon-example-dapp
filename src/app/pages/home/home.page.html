Connect dApps with Wallets

Connect Beacon Download Extension Alpha - Beacon is in early Alpha development. Not every component is fully functional and not all flows have been implemented yet. Reach out with questions.

Beacon is the implementation of tzip-10 which describes an interaction standard between a wallet and a dApp.

Where a dApp impelementing the beacon-sdk can build up a channel and send messages over a peer to peer communication layer to a wallet using the sdk. This allows for a communication for example of a mobile wallet with a desktop application. The requests of the dApp are sent to the wallet, signed and returned to the application.

Approach

For the user there are two option to interact with dApps.

Either a browser extension is installed that communicates with the dApp and wallet by only pairing them once or each dApp is paired directly with the wallet.

Browser Extension

The Beacon browser extension is responsible for the communication with the dApp and the wallet.

dApp Integration

With the implementation of the beacon-sdk, the dApp can also support direct connections. The user has to scan the pairing qr with his wallet for each dApp.

Transport Layer

As specified in tzip-10 Beacon utilizes a peer to peer network built with the matrix protocol to transport messages from the dApps to the Wallets thus leveraging a decentralized approach.

Components

The various Beacon components repositories for developers.

SDK

The beacon-sdk simplifies and abstracts the communication between dApps and wallets over different transport layers.

Extension

The Beacon Extension implements the beacon-sdk to interact with dApps that have implemented the beacon-sdk and use the various signing methods.

dApp

The Beacon Example dApp is the reference implementation of a dApp that uses the beacon-sdk with various examples of the different message types.
