Beacon is the implementation of tzip-10 which describes an interaction standard between a wallet and a dApp.

Where a dApp impelementing the beacon-sdk can build up a channel and send messages over a peer to peer communication layer to a wallet using the sdk. This allows for a communication for example of a mobile wallet with a desktop application. The requests of the dApp are sent to the wallet, signed and returned to the application.

Transport Layer

As specified in tzip-10 Beacon utilizes a peer to peer network built with the matrix protocol to transport messages from the dApps to the Wallets thus leveraging a decentralized approach.

Components

The various Beacon components repositories for developers.

SDK

The beacon-sdk simplifies and abstracts the communication between dApps and wallets over different transport layers.

Android SDK

The beacon-android-sdk simplifies and abstracts the communication between dApps and wallets for Android.

iOS SDK

The beacon-android-ios simplifies and abstracts the communication between dApps and wallets for iOS.

Example dApp

The Beacon Example dApp is a reference implementation of a dApp that uses the beacon-sdk with various examples of the different message types.

Spire

Spire implements the beacon-sdk to interact with dApps that have implemented the beacon-sdk and use the various signing methods.
